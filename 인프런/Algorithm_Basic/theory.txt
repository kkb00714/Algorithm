그림으로 쉽게 배우는 자료구조와 알고리즘 (기본편)

    ※ 자료구조 : 데이터가 어떤 구조로 저장되고 어떻게 사용되는지를 나타냄
        ex) 변수, 자료구조, 등

    ※ 자료구조에 따라 처리 방법이 달라짐

    ※ 알고리즘 : 컴퓨터가 따라할 수 있도록 문제를 해결하는 절차나 방법을 자세히 설명하는 과정.

    ※ 시간 복잡도 : 특정 알고리즘이 어떤 문제를 해결하는 데 걸리는 시간
        => Big - Ω : 최선의 경우 한 번에 찾음
        Big - O : 최악의 경우 배열의 길이만큼 걸림
        Big - θ : 평균의 경우 배열의 길이 중간 만큼 걸림
    
        ex) Big - O 사용시 : 계산량이 얼마나 늘어나는지를 표현하는 방법
            => 배열의 첫번째 원소부터 가장 마지막 원소까지 비교하면서 찾는 경우,
            i) 최악의 경우 찾는 데이터가 배열의 끝에 있을 때
                => 데이터를 n 번 만에 찾을 수 있는데 이를 O(n) 으로 표현
                => 데이터가 많아지면 그에 비례해서 계산량이 많아지므로, O(n)의 알고리즘은 선형시간 알고리즘이라고 부름

            ii) 상수시간 알고리즘   
                => O(1) 으로 표현. 
                => 입력의 크기에 상관없이 상수의 시간이 걸린다는 의미 (계산량이 꼭 한번일 이유는 없음.)
                => 

            iii) Big - O 표기법의 특징
                - 빅 오 표기법으로 표기할 때에는 상수는 큰 의미가 없으므로 제곱
                ex) n^2 + 2n + 100의 성능을 보이는 알고리즘 => O(n^2),
                3n^2 + 100n의 성능을 보이는 알고리즘 => O(3n^2) => (상수까지 마저 제거해준다면) => O(n^2)
